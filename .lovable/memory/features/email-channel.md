# Memory: features/email-channel
Updated: now

نظام إدارة قنوات البريد الإلكتروني (SMTP/IMAP) يوفر تجربة احترافية مشابهة لـ Outlook؛ تدعم حسابات متعددة مع ميزة 'اختبار الاتصال' وأدلة للـ App Passwords. تتميز الواجهة بنظام 'الرسائل الاحترافية' حيث يظهر هيدر منفصل (الموضوع، المرسل، المستلم، والـ CC) أعلى كل رسالة، مع دعم كامل للـ CC و BCC في الإرسال والاستقبال. تدعم واجهة إنشاء الرسائل نظام 'البطاقات' (Chips) لإضافة مستلمين متعددين في حقول 'To' و 'Cc' و 'Bcc' (بالضغط على Enter أو فاصلة).

## البنية التحتية للبيانات
- **جدول `email_message_details`**: جدول مخصص على القاعدة الخارجية لتخزين البيانات الوصفية للإيميلات بأعمدة مستقلة (بدلاً من JSON metadata)، يشمل: الموضوع، المرسل، المستلم، CC، BCC، Message-ID، In-Reply-To، References، المرفقات، واتجاه الرسالة (inbound/outbound).
- **التوافق العكسي**: يقرأ النظام من `email_message_details` أولاً ثم يعود لـ `metadata` JSON للرسائل القديمة.
- **إنشاء الجدول**: عبر `init-external-email-table` Edge Function التي تدعم الاتصال المباشر (DB URL) والـ SQL HTTP API.

## المرفقات (Attachments)
- دعم إرسال مرفقات عبر SMTP باستخدام base64 encoding
- عرض المرفقات كبطاقات في فقاعة الرسالة مع أيقونة 📎
- تخزين أسماء المرفقات في `email_message_details.email_attachments` (JSONB)

## الميزات الأساسية
يتم جلب الرسائل كل دقيقتين عبر عميل IMAP مخصص (RawImapClient) يدعم التشفير ومعالجة النصوص العربية بترميز MIME واستخراج المحتوى من صيغ Base64 و HTML. يتم تجميع الرسائل في خيوط (Threads) بناءً على مراجع الرد (In-Reply-To/References) أو المواضيع الموحدة، مع استخدام Message-ID لمنع التكرار. في الإرسال، يتم ربط الردود بالثريد الأصلي وإضافة 'Re:' تلقائياً. كما يتميز النظام بتبويب فلتر مخصص 📧، وتوجيه آلي للموظفين/الفرق، وسلوك لوحة مفاتيح مخصص (Enter للإرسال و Shift+Enter للسطر الجديد)، مع تعطيل أدوات الواتساب غير المتوافقة لضمان نظافة الواجهة.
