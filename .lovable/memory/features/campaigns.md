# Memory: features/campaigns
Updated: now

نظام حملات متطور يدعم:
1. **نظام الطابور (Queue System)**: عند إرسال حملة، يتم تقسيم المستلمين إلى دفعات (500 رسالة لكل دفعة) وإدخالها في جدول `campaign_queue`. يتم معالجة الدفعات تلقائياً عبر Cron Job كل دقيقة مع تحكم بالتزامن (3 حملات كحد أقصى بنفس الوقت).
2. الحملات المتكررة (Recurring): جدولة يومية، أسبوعية، أو شهرية مع تحديث تلقائي للجمهور المستهدف.
3. تتبع الأداء اللحظي: تحديث حالات الرسائل (تم التوصيل، تمت القراءة، فشل) فوراً بناءً على ويب هوك ميتا.
4. إدارة الفشل: عرض أسباب الفشل وإمكانية 'إعادة إرسال الرسائل الفاشلة' بعد مهلة 24 ساعة.
5. تكامل Excel: توليد قوالب ذكية تحتوي على أعمدة المتغيرات (Variables) تلقائياً بناءً على القالب المختار، ودعم رفع وتصدير ملفات XLSX.
6. حاسبة التكلفة: تقدير تكلفة الحملة بناءً على أسعار ميتا المحدثة للسعودية (Marketing ~0.35 SAR / Utility ~0.15 SAR) مع روابط للمصدر الرسمي.
7. **حالات الحملة**: draft → queued (تم التقسيم) → sending (جاري الإرسال) → sent (مكتمل) / paused (حدود الإرسال) / failed.
8. **حماية من الانهيار**: send-campaign يقسّم فقط ولا يُرسل مباشرة. process-campaign-queue هو المسؤول عن الإرسال الفعلي بشكل تدريجي.
